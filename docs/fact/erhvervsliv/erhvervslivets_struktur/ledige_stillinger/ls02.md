table: fact.ls02
description: Ledige stillinger efter region, enhed og tid
measure: indhold (unit -)
columns:
- region: join dim.nuts on region=kode [approx]; levels [1, 2]
- enhed: values [LS=Ledige stillinger (antal), ALS=Andel ledige stillinger (procent)]
- tid: date range 2010-01-01 to 2024-01-01
dim docs: /dim/nuts.md

notes:
- Annual data (one row per year). For quarterly, use lsk02 instead.
- enhed is a measurement selector: LS=count, ALS=percentage. Always filter to one. ALS must never be summed.
- region joins dim.nuts but the join is partially misleading. Code 1 in dim.nuts is "Landsdel Byen København" (niveau 2), but in ls02 code 1 is the national total (verified: it equals the sum of the 5 regions + 135 across all years). Do NOT treat code 1 as a landsdel when querying this table.
- Only codes 81-85 join correctly to dim.nuts niveau 1 (the 5 Danish regions). Use these for regional breakdown.
- Code 135 is unmatched in dim.nuts — a special/residual category with small but non-zero vacancy counts.
- To get the 5-region breakdown: WHERE region IN ('81','82','83','84','85') AND enhed='LS', then JOIN dim.nuts ON region=kode to get region names.
- ColumnValues("nuts", "titel", for_table="ls02") will show the 7 region codes (including the misleading code 1 and unmatched 135).
- Map: /geo/regioner.parquet — merge on region=dim_kode. Filter to region IN ('81','82','83','84','85') only (exclude code 1 and 135).
