table: fact.gf13
description: Generel firmastatistik efter branche (DB07 19-grp), enhed, firmastørrelse (fuldtidsansatte) og tid
measure: indhold (unit -)
columns:
- branchedb0738: join dim.db on branchedb0738=kode::text
- enhed: values [AFI=Firmaer (antal), AFU=Fuldtidsansatte (antal), OMS=Omsætning (mio kr.) ]
- firmstr: values [TOT=I alt, 0000=Ingen ansatte, 010=Under 10 ansatte, 101=10-49 ansatte, 102=50-249 ansatte, 103=250 ansatte og derover]
- tid: date range 2019-01-01 to 2023-01-01
dim docs: /dim/db.md

notes:
- branchedb0738 uses DB07 19-group letter codes (A–S, TOT, X). These do NOT match dim.db which uses numeric codes. Treat branchedb0738 as fully inline — use ColumnValues("gf13", "branchedb0738") for the full list with labels. No dim join needed or possible.
- enhed is a measurement selector (AFI=firmaer, AFU=fuldtidsansatte, OMS=omsætning). Always filter to exactly one value — every combination of branchedb0738+firmstr is repeated once per enhed value.
- firmstr includes TOT (I alt) plus 5 size buckets (Ingen ansatte, Under 10, 10–49, 50–249, 250+). Filter to TOT for totals or to specific buckets; never sum all firmstr values together.
