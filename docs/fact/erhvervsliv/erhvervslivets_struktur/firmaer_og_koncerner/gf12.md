table: fact.gf12
description: Generel firmastatistik efter kommune, branche (DB07 10 19 127-grp. og detaljeret) og tid
measure: indhold (unit Antal)
columns:
- komk: join dim.nuts on komk=kode; levels [1, 2, 3]
- branchedb0710til127: join dim.db on branchedb0710til127=kode::text; levels [2, 3, 5]
- tid: date range 2019-01-01 to 2023-01-01
dim docs: /dim/db.md, /dim/nuts.md

notes:
- komk joins dim.nuts. Code 0 = "Hele landet" (aggregate, not in dim). dim.nuts has niveau 1 = landsdelene, niveau 2 = landsdele-subdivisions, niveau 3 = kommuner. For kommune-level queries use `WHERE d.niveau = 3`. ColumnValues("nuts", "titel", for_table="gf12") shows the 114 matched codes.
- branchedb0710til127 mixes 10-group codes (1–11, matching dim.db niveau 2), 127-group codes without leading zeros (e.g. 51000, 13000, matching dim.db niveau 5), and very detailed 5–6 digit codes with leading zeros (e.g. 011100, 011300 = NACE subgroup codes) that do NOT match dim.db. The 79% dim.db match rate reflects the subset of codes that use the path-based numeric scheme.
- For the unmatched detailed codes (e.g. 011100 = "Dyrkning af etårige afgrøder"), use ColumnValues("gf12", "branchedb0710til127") to get labels — no dim.db join available for zero-padded NACE codes.
- indhold is always a firm count (Antal) — no enhed selector to filter here, unlike gf13/gf14.
- Map: /geo/kommuner.parquet (niveau 3), /geo/landsdele.parquet (niveau 2), or /geo/regioner.parquet (niveau 1) — merge on komk=dim_kode. Exclude komk=0.
