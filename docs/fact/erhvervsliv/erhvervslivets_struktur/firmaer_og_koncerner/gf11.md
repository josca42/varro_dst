notes:
- maengde4 is a measurement selector (AFI=firmaer, AFU=fuldtidsansatte, OMS=omsætning). Always filter to exactly one value — leaving it open triples every count.
- branchedb071019127 mixes all three granularity levels in a single column. Use the code format to distinguish: letter codes (A–S, X) = 19-group; 1–2 digit numbers (1–11) = 10-group; 5-digit codes (e.g. 13000, 51000) = 127-group. TOT appears only at the aggregate level.
- The dim.db join works for 10-group numeric codes (1–11, matching dim.db niveau 2) and some 127-group codes, but NOT for the 19-group letter codes. For letter codes, use ColumnValues("gf11", "branchedb071019127") which returns all 160 codes with labels — no dim join needed.
- To work at a single granularity level, add a WHERE filter: `branchedb071019127 ~ '^[A-Z]$'` for 19-group sections; `branchedb071019127 ~ '^\d{1,2}$'` for 10-group; `branchedb071019127 ~ '^\d{5}'` for 127-group. Including mixed levels silently sums across overlapping aggregates.
- 127-group codes with leading zeros (e.g. 01000, 02000) do NOT match dim.db — they use a NACE-based scheme with leading zeros, while dim.db uses a path-based numeric scheme (e.g. 13000 = within-section group 1.3). Stick to ColumnValues for labels at this level.
