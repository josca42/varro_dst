table: fact.bdf307
description: Beskæftigede i landbrug og gartneri efter område, enhed, type, arbejdstid og tid
measure: indhold (unit Antal)
columns:
- omrade: join dim.nuts on omrade=kode [approx]; levels [1, 2]
- enhed: values [ANTAL=Bedrifter (antal), PERS=Antal personer]
- type: values [PB=Personlig bruger, B=Bestyrer, MAEF=Medarbejdende ægtefælle, FM=Familie medhjælp, FFM=Fast fremmed medhjælp, MIALT=Medhjælp i alt, BIALT=Beskæftigede i alt]
- arbejdstid: values [8=1-8 t./uge, 18=9-18 t./uge, 26=19-26 t./uge, 36=27-36 t./uge, 37OV=37t./uge eller derover, IALT=I alt]
- tid: date range 2007-01-01 to 2023-01-01
dim docs: /dim/nuts.md

notes:
- omrade joins dim.nuts with the same 12 codes as bdf11: niveau 1 (81-85), niveau 2 (4,7,8,9,10), plus aggregates 0 and 15. See bdf11 notes for join details.
- enhed is a measure-type selector: ANTAL (count of bedrifter) vs PERS (count of persons). These give different results for aggregate types. Always filter to one.
- type contains individual worker categories (PB, B, MAEF, FM, FFM) plus two aggregate codes: MIALT (medhjælp i alt = FM+FFM+MAEF) and BIALT (all workers including PB+B+MIALT). Never sum across all type values — use BIALT for the grand total or individual types for breakdown.
- arbejdstid contains weekly hours brackets (8,18,26,36,37OV) plus IALT (total). Filter to IALT to get counts regardless of hours worked. Don't sum across brackets AND IALT simultaneously.
- For total employed persons nationally: WHERE enhed='PERS' AND type='BIALT' AND arbejdstid='IALT' AND omrade=0 (gives ~71,000 in 2023).
- For farm-level (bedrifter) count instead of person count: use enhed='ANTAL'.
- Map: /geo/regioner.parquet (niveau 1, omrade 81-85) or /geo/landsdele.parquet (niveau 2, omrade 4,7,8,9,10 — 5 of 11 landsdele only) — merge on omrade=dim_kode. Exclude omrade IN (0, 15).
