table: fact.ipop2021
description: Industriens produktionsindeks efter sæsonkorrigering, branche (DB07) og tid
measure: indhold (unit Indeks)
columns:
- saeson: values [EJSÆSON=Ikke sæsonkorrigeret, SÆSON=Sæsonkorrigeret]
- branche07: join dim.nr_branche on branche07=kode [approx]; levels [1, 2, 3, 4]
- tid: date range 2000-01-01 to 2025-08-01
dim docs: /dim/nr_branche.md

notes:
- saeson is a measurement selector: every branche07/tid combination exists for both EJSÆSON and SÆSON. Always filter to one — e.g. WHERE saeson = 'EJSÆSON' for raw series or saeson = 'SÆSON' for seasonally adjusted. Forgetting this doubles all counts.
- indhold is a production index (base=100), not an absolute value. Summing across branches is meaningless — use values for trend/level comparison.
- branche07 uses a mixed coding scheme. Standard DB07 letter codes (B, C, CA, CB, CC, CF, CG, CH, CI, CJ, CK, CL, CM, D) match dim.nr_branche at niveau 3. Custom codes NOT in the dim table: numeric sub-codes (10001–10005 for CA sub-sectors, 22000, 23001, 23002, 28001, 28002, 29000, 30000 for CG/CH/CK/CL sub-sectors), aggregate combos (BC, BCD, CDE, C30, C31, C32), and MIG sector groups (S1, S11, S12, S2, S3, S4, S41, S5).
- ipop2021 has 42 branche07 codes vs 39 in ipoo2021; it adds BCD (Råstofindvinding, industri og energiforsyning), D (Energiforsyning), and S5 (Fremstilling af energiprodukter og energiforsyning).
- Use ColumnValues("ipop2021", "branche07") to browse all available industries with labels — the parquet covers all custom codes that the dim join misses.
- If joining dim.nr_branche, B and C match at niveau 1, 2, AND 3 — always add AND d.niveau = 3 to avoid triple-counting.
