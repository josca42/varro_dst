table: fact.ipoo2021
description: Industriens omsætningssituation efter omsætning, sæsonkorrigering, branche (DB07) og tid
measure: indhold (unit Indeks)
columns:
- omsaetning: values [410=Omsætning i løbende priser, 411=Eksportomsætning i løbende priser, 415=Hjemmemarkedsomsætning i løbende priser]
- saeson: values [EJSÆSON=Ikke sæsonkorrigeret, SÆSON=Sæsonkorrigeret]
- branche07: join dim.nr_branche on branche07=kode [approx]; levels [1, 2, 3, 4]
- tid: date range 2000-01-01 to 2025-08-01
dim docs: /dim/nr_branche.md

notes:
- saeson is a measurement selector: every branche07/tid combination exists for both EJSÆSON and SÆSON. Always filter to one — e.g. WHERE saeson = 'EJSÆSON' for raw series or saeson = 'SÆSON' for seasonally adjusted. Forgetting this doubles all counts.
- omsaetning is also a measurement selector: values 410 (total), 411 (eksport), 415 (hjemmemarked). All three are present for every row — filter to exactly one. 410 is total turnover; 411 and 415 are sub-components (do not sum them with 410).
- indhold is an index (base=100), not an absolute value. Summing across branches is meaningless — use the index values for trend comparison.
- branche07 uses a mixed coding scheme. Standard DB07 letter codes (CA, CB, CC, CF, CG, CH, CI, CJ, CK, CL, CM, B, C) match dim.nr_branche at niveau 3. But many codes are custom and NOT in the dim table: numeric sub-codes (10001–10005 = sub-sectors of CA fødevarer; 22000, 23001, 23002, 28001, 28002, 29000, 30000 = sub-sectors within CG/CH/CK/CL), aggregate combos (BC, CDE, C30, C31, C32), and MIG sector groups (S1, S11, S12, S2, S3, S4, S41).
- Use ColumnValues("ipoo2021", "branche07") rather than the dim join — the parquet includes full labels for all codes including the custom ones not in dim.nr_branche.
- If you do join dim.nr_branche, B and C match at niveau 1, 2, AND 3, so always add AND d.niveau = 3 to avoid triple-counting. Only 15 of 39 codes match the dim table at all.
- Typical query pattern: SELECT f.branche07, f.indhold FROM fact.ipoo2021 f WHERE f.omsaetning = '410' AND f.saeson = 'EJSÆSON' AND f.tid = '2024-01-01' AND f.branche07 IN ('CA','CB','CC','CF','CG','CH','CI','CJ','CK','CL','CM')
