table: fact.ej99
description: Prisindeks for andelsboliger og ejerboliger efter område, boligtype, enhed og tid
measure: indhold (unit -)
columns:
- omrade: join dim.nuts on omrade=kode; levels [2]
- boltyp: join dim.ejendom on boltyp=kode [approx: Missing codes 2104 and 100 in ejendom dimension (has 2103 instead of 2104)]; levels [1, 2]
- enhed: values [100=Indeks, 210=Ændring i forhold til kvartalet før (pct.), 310=Ændring i forhold til samme kvartal året før (pct.)]
- tid: date range 2015-01-01 to 2025-04-01
dim docs: /dim/ejendom.md, /dim/nuts.md

notes:
- `boltyp` is NOT joinable to dim.ejendom despite the approx link. The codes are specific to this table: 100=Andelsboliger, 111=Enfamiliehuse, 2104=Ejerlejlighed. Code 2104 does NOT exist in dim.ejendom (which uses 2103 for ejerlejligheder). Use ColumnValues("ej99", "boltyp") — do not attempt a dim.ejendom join.
- `enhed` is a measurement selector: 100=Indeks, 210=QoQ%, 310=YoY%. Always filter to one value — omitting this triples the result.
- `omrade` joins dim.nuts niveau 2 only (11 landsdele, kode 1-11). Code 0 = "Hele landet" (national total, not in dim). No region-level or municipality-level breakdown in this table.
- Only table covering andelsboliger (code 100) alongside ejerboliger — unique in this subject.
- Quarterly from 2015 only (short series compared to other tables).
- Map: /geo/landsdele.parquet (niveau 2 only) — merge on omrade=dim_kode. Exclude omrade=0.
