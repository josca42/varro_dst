table: fact.ej121
description: Sæsonkorrigerede ejendomssalg, almindelig fri handel efter region, ejendomskategori, enhed og tid
measure: indhold (unit -)
columns:
- region: join dim.nuts on region=kode; levels [1]
- ejkat20: join dim.ejendom on ejkat20=kode; levels [1, 2]
- tal: values [202=Prisindeks, sæsonkorrigeret, 362=Prisindeks, ændring i forhold til kvartalet før (pct.), 364=Prisindeks, ændring i forhold til samme kvartal året før (pct.), 204=Antal salg, sæsonkorrigeret, 368=Antal salg, Ændring i forhold til kvartalet før (pct.), 370=Antal salg, Ændring i forhold til samme kvartal året før (pct.)]
- tid: date range 2005-01-01 to 2025-04-01
dim docs: /dim/ejendom.md, /dim/nuts.md

notes:
- `tal` encodes TWO distinct measures combined with time-comparison variants. Group 1: Prisindeks (202=level, 362=QoQ%, 364=YoY%). Group 2: Antal salg (204=level, 368=QoQ%, 370=YoY%). Always filter to exactly one `tal` value — mixing them would mix prices and counts into one result.
- `region` joins dim.nuts niveau 1 only (5 regioner, kode 81-85). Code 0 = "Hele landet" (national total, not in dim). This is the only geographic breakdown — no landsdele or kommuner.
- `ejkat20` joins dim.ejendom at both niveau 1 and 2 (only 3 property types in this table). Filter `d.niveau` to pick one level.
- All data is seasonally adjusted (sæsonkorrigeret) — not comparable to raw counts in ejen77/ejen88. Covers only "almindelig fri handel" (ordinary free-market transactions).
- Quarterly from 2005. Only table with both seasonally-adjusted price index AND seasonally-adjusted sales counts by region.
- Map: /geo/regioner.parquet (niveau 1 only) — merge on region=dim_kode. Exclude region=0.
