table: fact.pris4221
description: Producentprisindeks for varer efter branche standardgrupperinger, enhed og tid
measure: indhold (unit Indeks)
columns:
- standgrp: join dim.db on standgrp=kode::text [approx]; levels [5]
- tal: values [100=Indeks, 200=Ændring i forhold til måneden før (pct.), 300=Ændring i forhold til samme måned året før (pct.)]
- tid: date range 2000-01-01 to 2025-09-01
dim docs: /dim/db.md

notes:
- tal is a measurement selector that triples the data — always filter to one value: tal='100' (Indeks), tal='200' (month-on-month %), tal='300' (year-on-year %).
- Do NOT use the dim.db join for standgrp. The match rate is only 2% and those matches are false positives — dim.db kode=22000 at niveau 5 is "Skovning" (forestry) while pris4221's standgrp='22000' means "Plast- og gummiindustri". The codes come from entirely different classification systems.
- standgrp uses a Danish producer-price-specific grouping: 5-digit codes for detailed industry sub-groups (10001=Slagterier, 10002=Fiskeindustri, 10003=Mejerier, etc.) plus letter-based NACE aggregates (C=Industri, CA=Føde-, drikke- og tobaksvareindustri, CB=Tekstil- og læderindustri, etc.) and MIG S-codes (S1–S5).
- Use ColumnValues("pris4221", "standgrp") to browse all 43 groups with labels. No dim join needed.
- This is the producer price index (only domestic production, no imports). For a version with both producer and import prices plus market split, use pris4321. From 2000 onwards.
