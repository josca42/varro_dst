table: fact.fu19
description: Husstandenes gennemsnitlige indkomst og forbrug efter indkomsttype og tid
measure: indhold (unit Kr. pr. husstand)
columns:
- indkomsttype: values [400=A. Erhvervsindkomst (B+C), 410=B. Løn mv., 419=C. Virksomhedsoverskud mv., 440=D. Formueindkomster, 450=E. Overførsler fra private ... 591=U. Pensions- og ATP-bidrag (eget og arbejdsgivers), 595=V. Indbetaling til privat livsforsikring mv., 596=X. Værdi af til- og ombygning mv. af bolig, 597=Y. Opsparing i øvrigt inkl. residual, 600=Z. Forbrug (P-Q-R-S-T)]
- tid: date range 2015-01-01 to 2023-01-01

notes:
- indkomsttype is a hierarchical accounting structure — never sum across all values. Some are sums of others: 400=410+419 (Erhvervsindkomst=Løn+Virksomhedsoverskud), and 400+440+450+480+495 = 499 (samlet indkomst). Summing all indkomsttype values gives a meaningless result.
- Code 600 (Z. Forbrug) = total household consumption expenditure, directly comparable to konsumgrp "00" (total) in fu12-fu18.
- No overcounting risk from category columns — each indkomsttype row is a distinct line. Just pick the line(s) you need.
- Useful for income-consumption context: compare 499 (samlet indkomst) with 600 (forbrug) to derive the savings rate over time.
