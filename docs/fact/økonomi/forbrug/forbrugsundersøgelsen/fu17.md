table: fact.fu17
description: Forbrug efter forbrugsgruppe, region, prisenhed og tid
measure: indhold (unit Kr. pr. husstand)
columns:
- konsumgrp: join dim.ecoicop on konsumgrp=kode::text [approx]; levels [1, 2]
- region: values [2001=Gennemsnitshusstand, 6010=Region Hovedstaden, 6020=Region Sjælland, 6030=Region Syddanmark, 6040=Region Midtjylland, 6050=Region Nordjylland]
- prisenhed: values [AARPRIS=Løbende priser, 08PRIS=Faste priser]
- tid: date range 2015-01-01 to 2023-01-01
dim docs: /dim/ecoicop.md

notes:
- prisenhed MUST be filtered (AARPRIS or 08PRIS) — doubles all rows.
- region must be filtered to one value: 2001=Gennemsnitshusstand, 6010=Region Hovedstaden, 6020=Region Sjælland, 6030=Region Syddanmark, 6040=Region Midtjylland, 6050=Region Nordjylland. Only regional breakdown in the fu series — no municipality level.
- konsumgrp: same ECOICOP dotted decimal scheme as fu13 (niveau 1 and 2, 62 codes). Same join formula and unmatched codes.
- Map: /geo/regioner.parquet — region uses custom codes (6010→84, 6020→85, 6030→83, 6040→82, 6050→81). Map with a CASE or lookup then merge on dim_kode. Exclude region=2001.
