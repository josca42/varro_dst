table: fact.np28
description: Produktivitetsudviklingen, KLEMS efter branche, type, prisenhed og tid
measure: indhold (unit Pct.)
columns:
- branche: join dim.nr_branche on branche=kode [approx]
- type: values [K_LP=Produktion pr. time, K_IT=It-kapitalintensitet (KLEMS), K_AK=Anden kapitalintensitet (KLEMS), K_LQ=Uddannelsesniveau (KLEMS), K_E=Forbrug i produktionen af energi, K_M=Forbrug i produktionen af materialer, K_S=Forbrug i produktionen af tjenester, K_TF=Totalfaktorproduktivitet (KLEMS)]
- prisenhed: values [VPR_V=Årlig vækstrate i pct., VPR_G=Årlig vækstrate i pct., rullende 5-års gennemsnit]
- tid: date range 1967-01-01 to 2017-01-01
dim docs: /dim/nr_branche.md

notes:
- branche uses V-prefixed codes that do NOT directly match dim.nr_branche. Correct join: JOIN dim.nr_branche d ON d.kode = REPLACE(SUBSTRING(f.branche, 2), '_', '-').
- branche niveaux 1–4 are all present simultaneously. Filter WHERE d.niveau = N to pick one level.
- Three productivity-specific aggregate codes outside dim.nr_branche: PBY, PMIALT, PTJEN (same as np25/np25v).
- prisenhed is a measurement selector — always filter to one: VPR_V or VPR_G.
- type is the KLEMS decomposition of productivity: K_LP (production per hour, headline), K_IT, K_AK, K_LQ, K_E (energy), K_M (materials), K_S (services), K_TF (TFP). Like np25/np25v, do NOT sum across types. These are decomposition contributions.
- Data only runs to 2017 — this is the most dated of the four tables. Use np25 or np25v for more recent data. np28 adds the intermediate-input channels (energy/materials/services) from the KLEMS framework that np25 lacks.
- For the most recent productivity decomposition (post-2017), use np25. For KLEMS including energy/materials/services: np28 (up to 2017 only).
