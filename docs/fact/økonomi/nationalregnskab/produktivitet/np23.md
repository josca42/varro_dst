table: fact.np23
description: Arbejdsproduktivitet efter branche, prisenhed og tid
measure: indhold (unit -)
columns:
- branche: join dim.nr_branche on branche=kode [approx]
- prisenhed: values [LPR_I=Indeks (2020=100), 2020-priser, kædede værdier, LPR_V=Årlig vækstrate i pct., 2020-priser, kædede værdier, LPR_G=Årlig vækstrate i pct., 2020-priser, kædede værdier, rullende 5-års gennemsnit]
- tid: date range 1966-01-01 to 2024-01-01
dim docs: /dim/nr_branche.md

notes:
- branche uses V-prefixed codes that do NOT directly match dim.nr_branche. Correct join: JOIN dim.nr_branche d ON d.kode = REPLACE(SUBSTRING(f.branche, 2), '_', '-'). This handles letter codes (VA→A, VC→C), numeric codes (V01000→01000), and combined codes (VD_E→D-E, VG_I→G-I, VM_N→M-N, VR_S→R-S).
- This table has the deepest branch detail: niveaux 1–5 are all present simultaneously. Always filter WHERE d.niveau = N to pick one level. np25/np25v/np28 only go to niveau 4.
- Four productivity-specific aggregate codes appear outside dim.nr_branche: PIALT (samlet økonomi — total economy, only in this table), PBY (erhverv/private business), PMIALT (markedsbestemt erhverv/market economy), PTJEN (tjenester/services). Filter to these directly by branche code without a dim join.
- prisenhed is a measurement selector that repeats every branch×year triple times. Always filter to one value: LPR_I (index, 2020=100), LPR_V (annual growth rate pct.), LPR_G (annual growth rate pct., rolling 5-year avg). Forgetting this silently triples all counts.
- This is the only table in the subject with an absolute level (LPR_I index). Use it for level comparisons; use np25 for decomposed growth rates.
