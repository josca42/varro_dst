table: fact.ky04
description: Kontanthjælp (personer og beløb) efter område, enhed, visitation og tid
measure: indhold (unit -)
columns:
- omrade: join dim.nuts on omrade=kode [approx]; levels [1, 2, 3]
- tal: values [1=Berørte personer (antal), 3=Udbetalt beløb (1.000 kr.)]
- visitation: values [TOT=I alt, 1=Jobparat, 2=Aktivitetsparat, 3=Åbenlyst uddannelsesparat, 4=Uddannelsesparat, 9=Uoplyst]
- tid: date range 2007-01-01 to 2025-06-01
dim docs: /dim/nuts.md
notes:
- tal is a measurement selector: 1=Berørte personer (antal), 3=Udbetalt beløb (1.000 kr.). Always filter to one value.
- omrade joins dim.nuts on omrade=kode. 98% match. Two codes are NOT in dim.nuts: 0=Hele landet (national total), 997=Udlandet (living abroad). Exclude these or handle separately when joining.
- Use ColumnValues("nuts", "titel", for_table="ky04") to see the 114 regions available. niveau 1=5 regioner, niveau 2=11 landsdele, niveau 3=98 kommuner. Filter d.niveau to pick granularity.
- visitation has total (TOT) and 5 categories. Filter visitation='TOT' unless you want the breakdown.
- Example — recipients by kommune: SELECT d.titel, SUM(f.indhold) FROM fact.ky04 f JOIN dim.nuts d ON f.omrade=d.kode AND d.niveau=3 WHERE f.tal='1' AND f.visitation='TOT' AND f.tid='2024-01-01' GROUP BY d.titel.
- Map: /geo/kommuner.parquet (niveau 3), /geo/landsdele.parquet (niveau 2), or /geo/regioner.parquet (niveau 1) — merge on omrade=dim_kode. Exclude omrade=0.
