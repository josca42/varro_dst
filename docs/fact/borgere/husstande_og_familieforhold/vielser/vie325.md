table: fact.vie325
description: Vielser efter mandens eller ældste persons bopæl, Kvindens eller yngste persons bopæl, vielsesmåned, vielsesmyndighed og tid
measure: indhold (unit Antal)
columns:
- bopaegt1: join dim.nuts on bopaegt1=kode; levels [3]
- bopaegt2: join dim.nuts on bopaegt2=kode; levels [3]
- vimdr: values [1=Januar, 2=Februar, 3=Marts, 4=April, 5=Maj, 6=Juni, 7=Juli, 8=August, 9=September, 10=Oktober, 11=November, 12=December, 999=Uoplyst måned]
- vimyn: values [1=Kirkelig, 2=Borgerlig, 3=Gift i udlandet, 4=Uoplyst vielsesmyndighed]
- tid: date range 2007-01-01 to 2024-01-01
dim docs: /dim/nuts.md

notes:
- `bopaegt1` and `bopaegt2` join dim.nuts at niveau 3 only (99 kommuner). Same coding as vie225: '0'="I alt" marginal total, '997'=abroad/unknown. See vie225 notes for details on marginal total usage.
- `vimdr` uses plain integers 1–12 (no TOT, no zero-padding). This differs from vie9/viedag which use '001'–'012' and TOT. There is no '999' in this table's vimdr — all months are identified.
- `vimyn` uses plain integers 1–4 (no TOT). There is no aggregate total for either vimdr or vimyn in this table.
- To get annual total: `WHERE bopaegt1='0' AND bopaegt2='0' AND tid=...` then SUM(indhold) across all 12 months × 4 authorities. Verified: 31582 for 2023.
- Best table for "which kommuner have most Kirkelig vs Borgerlig vielser" type questions — unique combination of regional + month + authority breakdown.
- Map: /geo/kommuner.parquet — merge on bopaegt1=dim_kode or bopaegt2=dim_kode (niveau 3). Fix the other bopæl column to '0' (marginal total) to get per-kommune totals. Exclude bopaegt=0 and bopaegt=997 from the mapped column.
