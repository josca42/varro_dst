table: fact.bol103
description: Boliger efter område, beboertype, anvendelse, antal værelser, boligstørrelse i kvm., husstandsstørrelse og tid
measure: indhold (unit Antal)
columns:
- amt: join dim.nuts on amt=kode; levels [1, 2, 3]
- bebo: values [1000=Boliger med CPR tilmeldte personer (beboede boliger), 2000=Boliger uden CPR tilmeldte personer (ubeboede boliger), 5000=Fritidshuse uden CPR tilmeldte personer (ubeboede fritidshuse)]
- anvendelse: values [125=Parcel/Stuehuse, 130=Række-, kæde- og dobbelthuse, 140=Etageboliger, 150=Kollegier, 160=Døgninstitutioner, 565=Fritidshuse, 570=Andet]
- antvaer: values [11=1 værelse, 21=2 værelser, 31=3 værelser, 41=4 værelser, 51=5 værelser, 6-=6 værelser eller flere, UOPL=Uoplyst]
- boligstor: values [-50=- 50 kvm, 50-74=50-74 kvm, 75-99=75-99 kvm, 100-124=100-124 kvm, 125-149=125-149 kvm, 150-174=150-174 kvm, 175+=175 kvm og derover, UOPL=Uoplyst]
- husstor: values [1=1 person, 2=2 personer, 3=3 personer, 4=4 personer, 5=5 personer, 6=6 personer, 7-=7 personer og derover, 0=Ubeboet]
- tid: date range 2010-01-01 to 2025-01-01
dim docs: /dim/nuts.md

notes:
- amt joins dim.nuts. kode=0 (Hele landet) is not in dim — use WHERE amt=0 directly for national totals. niveau 1=5 regioner, 2=11 landsdele, 3=98 kommuner.
- husstor=0 means "Ubeboet" (unoccupied), which aligns with bebo=2000 or 5000. Don't mix occupied and unoccupied dwellings in the same sum unless intentional.
- No aggregate codes for antvaer, boligstor, husstor — SUM all values to get totals across those dimensions.
- boligstor buckets are: -50, 50-74, 75-99, 100-124, 125-149, 150-174, 175+, UOPL. No finer granularity than these bands.
- antvaer goes up to '6-' (6+ rooms), UOPL for unknown. Individual room counts only 1–6+.
- Map: /geo/kommuner.parquet (niveau 3), /geo/landsdele.parquet (niveau 2), or /geo/regioner.parquet (niveau 1) — merge on amt=dim_kode. Exclude amt=0.
