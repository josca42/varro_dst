table: fact.bol106
description: Boliger efter område, enhed, anvendelse og tid
measure: indhold (unit -)
columns:
- omrade: join dim.nuts on omrade=kode; levels [1, 2, 3]
- enhed: values [GNSAB=Gennemsnitlig areal pr. bolig (m2), GNSAP=Gennemsnitligt areal per person (m2), GNSPB=Gennemsnitlig personer pr. bolig (Antal)]
- anvendelse: values [TOT=I alt, 125=Parcel/Stuehuse, 130=Række-, kæde- og dobbelthuse, 140=Etageboliger, 150=Kollegier, 160=Døgninstitutioner, 565=Fritidshuse, 570=Andet]
- tid: date range 2010-01-01 to 2025-01-01
dim docs: /dim/nuts.md

notes:
- CRITICAL: enhed is a measurement selector. Every omrade/anvendelse/tid combination appears 3 times — once per enhed value. Always filter to exactly one enhed: GNSAB (gennemsnitlig areal pr. bolig, m²), GNSAP (gennemsnitligt areal per person, m²), or GNSPB (gennemsnitlig personer pr. bolig). Omitting this filter triples all counts and makes aggregation meaningless.
- indhold values are averages, not counts — do not SUM across regions to get a national average; use WHERE omrade=0 for Hele landet instead (kode=0 not in dim.nuts).
- anvendelse has a 'TOT' code here (total across all building types) — unlike bol101–105 which list only individual types. Use TOT to avoid having to SUM across all anvendelse values.
- omrade joins dim.nuts. niveau 1=5 regioner, 2=11 landsdele, 3=98 kommuner. ColumnValues("nuts", "titel", for_table="bol106") shows available regions.
- Map: /geo/kommuner.parquet (niveau 3), /geo/landsdele.parquet (niveau 2), or /geo/regioner.parquet (niveau 1) — merge on omrade=dim_kode. Exclude omrade=0.
