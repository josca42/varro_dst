table: fact.atr112
description: Arbejdstidsregnskab (kvartal) (sæsonkorrigeret) efter branche (DB07 10- og 19-grp), type, socioøkonomisk status og tid
measure: indhold (unit -)
columns:
- branchedb071038: join dim.db on branchedb071038=kode::text [approx]; levels [2, 3]
- type: values [HACS=Præsteret tid (1000 timer), ABESK=Beskæftigede (antal), AJOB=Job (antal), WSUM=Lønsum (mio. kr.)]
- socio: join dim.socio on socio=kode::text [approx]; levels [2]
- tid: date range 2008-01-01 to 2025-04-01
dim docs: /dim/db.md, /dim/socio.md

notes:
- Seasonally adjusted (sæsonkorrigeret) version of atr110. Use this table for trend analysis; use atr110 for unadjusted quarterly data.
- No `sektor` column — unlike atr110 which has sektor breakdown.
- `type` is a measurement selector: HACS=sæsonkorrigeret præsteret tid (1000 timer), ABESK, AJOB, WSUM. Every dimension combination appears 4x — always filter to one type.
- `branchedb071038` has identical coding to atr110's `branche07`: mixes numeric 1-11 (10-gruppe) and letters A-S, X (19-gruppe) in the same column. Always filter to ONE scheme. See atr110 notes for details on the join behavior with dim.db.
- `socio` same as atr110: code `14` not in dim.socio, only `13` joins.
