table: fact.atr118
description: Arbejdstidsregnskab på nationalregnskabets branchegrupperinger (år) efter branche (DB07), sektor, type, socioøkonomisk status, køn og tid
measure: indhold (unit -)
columns:
- branche07: join dim.db on branche07=kode::text [approx]; levels [5]
- sektor: values [TOT=I alt, 0=Virksomheder og organisationer, 1=Offentlig forvaltning og service]
- type: values [HAC=Præsteret tid (1000 timer), ABESK=Beskæftigede (antal), AJOB=Job (antal), WSUM=Lønsum (mio. kr.)]
- socio: join dim.socio on socio=kode::text [approx]; levels [2]
- kon: values [TOT=I alt, M=Mænd, K=Kvinder]
- tid: date range 2008-01-01 to 2023-01-01
dim docs: /dim/db.md, /dim/socio.md

notes:
- `type` is a measurement selector: HAC, ABESK, AJOB, WSUM. Always filter to one type.
- `branche07` uses nationalregnskabets (national accounts) industry classification — a different grouping from standard DB07. Two parallel coding series exist in the same column: 5-digit codes (01000, 02000...) and 6-digit codes (010000, 020000...) that represent IDENTICAL data values. Always filter to ONE scheme.
- Only 17 of 186 codes join dim.db (niveau 5 only, 9% match). Do NOT expect dim.db joins to work for most codes — this table's classification is purpose-built for national accounts, not for standard DB07 dim lookups.
- Also contains non-standard combined codes (e.g. 06090, 41430, 37390) representing national accounts groupings that cut across standard DB07 sections.
- `socio` at niveau 2: same as atr116 (11, 12, 13, TOT — all non-TOT join dim.socio).
- Use this table when analysing labor input in a national accounts context (GDP components, sector contributions). For standard industry breakdowns, prefer atr116.
