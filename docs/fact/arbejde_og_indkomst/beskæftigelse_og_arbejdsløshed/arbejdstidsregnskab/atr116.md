table: fact.atr116
description: Arbejdstidsregnskab (år) efter branche (DB07), sektor, type, socioøkonomisk status, køn og tid
measure: indhold (unit -)
columns:
- branche07: join dim.db on branche07=kode::text [approx]; levels [2, 3, 5]
- sektor: values [TOT=I alt, 0=Virksomheder og organisationer, 1=Offentlig forvaltning og service]
- type: values [HAC=Præsteret tid (1000 timer), ABESK=Beskæftigede (antal), AJOB=Job (antal), WSUM=Lønsum (mio. kr.)]
- socio: join dim.socio on socio=kode::text; levels [2]
- kon: values [TOT=I alt, M=Mænd, K=Kvinder]
- tid: date range 2008-01-01 to 2023-01-01
dim docs: /dim/db.md, /dim/socio.md

notes:
- `type` is a measurement selector: HAC, ABESK, AJOB, WSUM. Always filter to one type.
- `branche07` contains multiple hierarchy levels mixed in one column — pick exactly ONE level for any query:
  - `TOT`: grand total
  - Numeric 1-11: DB07 10-gruppe aggregations (1-10 groups + 11=uoplyst)
  - Letters A-S, X: DB07 19-gruppe (afsnit letter codes, e.g. A=Landbrug, C=Industri, Q=Sundhed)
  - Sub-letters CA-CM, JA-JC, MA-MC, QA-QB: sub-sections within C, J, M, Q — more detailed than letters
  - 5-digit codes ending in 000 (e.g. 11000, 25000): some join dim.db at niveau 5 (use ColumnValues("db", "titel", for_table="atr116") to see which ones); NACE-style codes with leading zeros (01000, 02000...) do NOT join dim.db
  - Mixing levels silently multiplies counts — C + CA + CB... would double-count industri
- 10 codes join dim.db at niveau 2, 1 at niveau 3, 13 at niveau 5. For niveau 2 joins add `d.niveau = 2`; code `11` is ambiguous in dim.db (matches both niveau 2 Drikkevarer and niveau 3 Etårige afgrøder).
- `socio` at niveau 2 only: 11=Selvstændige, 12=Medarbejdende ægtefælle, 13=Lønmodtagere, TOT. All three non-TOT join dim.socio. More detailed than atr110/112 which only have socio=13,14,TOT.
- `kon`: TOT, M, K. Filter to TOT or a specific gender — don't sum M+K+TOT.
- This is the most detailed annual table: has branche, sektor, socio, kon all together. Use when gender breakdown by industry is needed.
