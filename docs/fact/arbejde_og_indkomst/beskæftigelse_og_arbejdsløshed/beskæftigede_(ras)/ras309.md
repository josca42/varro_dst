table: fact.ras309
description: Beskæftigede efter branche (DB07), køn og tid
measure: indhold (unit Antal)
columns:
- branche07: join dim.db on branche07=kode::text; levels [5]
- kon: values [TOT=I alt, M=Mænd, K=Kvinder]
- tid: date range 2008-01-01 to 2023-01-01
dim docs: /dim/db.md
notes:
- branche07 uses the most detailed DB07 coding: 6-digit codes (011100, 101110...). Most codes (92%) match dim.db at niveau 5 (6-digit codes like 101110). The unmatched codes (011100, 011200...) are 4-digit codes zero-padded to 6 digits that represent niveau 4 — they don't directly map to dim.db. The ColumnValues file has all labels, so ColumnValues("ras309", "branche07") is the easiest way to browse and doesn't need a dim join.
- TOT is the only aggregate code — all other codes are at the detailed NACE level. No sector-level hierarchy available in this table; use ras311 for DB07-127 groupings.
- kon has TOT, M, K.
