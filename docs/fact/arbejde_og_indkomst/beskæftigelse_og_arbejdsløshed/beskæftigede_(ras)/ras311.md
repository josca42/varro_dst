table: fact.ras311
description: Beskæftigede (ultimo november) efter branche (DB07), herkomst, køn og tid
measure: indhold (unit Antal)
columns:
- branche07: join dim.db on branche07=kode::text; levels [2, 3, 5]
- herkomst: join dim.herkomst on herkomst=kode::text; levels [1]
- kon: values [TOT=I alt, 1=Mænd, 2=Kvinder]
- tid: date range 2008-01-01 to 2023-01-01
dim docs: /dim/db.md, /dim/herkomst.md
notes:
- branche07 uses DB07-127 coding with two hierarchy levels (same scheme as ras300): single/double-digit sector totals and 5-digit sub-sector codes. Does NOT join dim.db correctly. Use ColumnValues("ras311", "branche07") for labels and never mix hierarchy levels.
- herkomst does NOT join dim.herkomst — uses inline codes: TOT=I alt, 1=Dansk oprindelse, 24=Indvandrere fra vestlige lande, 25=Indvandrere fra ikke-vestlige lande, 34=Efterkommere fra vestlige lande, 35=Efterkommere fra ikke-vestlige lande. Use ColumnValues("ras311", "herkomst").
- kon uses unusual numeric coding: TOT=I alt, 1=Mænd, 2=Kvinder (not the usual M/K). Use these literal values in WHERE clauses.
