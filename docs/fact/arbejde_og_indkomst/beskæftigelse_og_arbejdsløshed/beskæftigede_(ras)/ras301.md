table: fact.ras301
description: Beskæftigede (ultimo november) efter område, branche (DB07), socioøkonomisk status, alder, køn og tid
measure: indhold (unit Antal)
columns:
- omrade: join dim.nuts on omrade=kode; levels [1, 2, 3]
- branche07: join dim.db on branche07=kode::text
- socio: join dim.socio on socio=kode
- alder: values [-15=Under 16 år, 16-19=16-19 år, 20-24=20-24 år, 25-29=25-29 år, 30-34=30-34 år, 35-39=35-39 år, 40-44=40-44 år, 45-49=45-49 år, 50-54=50-54 år, 55-59=55-59 år, 60-64=60-64 år, 65-66=65-66 år, 67-=67 år og derover]
- koen: values [M=Mænd, K=Kvinder]
- tid: date range 2008-01-01 to 2023-01-01
dim docs: /dim/db.md, /dim/nuts.md, /dim/socio.md
notes:
- omrade joins dim.nuts at 98%. Code 0=Hele landet and 950=Uden for Danmark are aggregate codes not in dim.nuts. Use ColumnValues("nuts", "titel", for_table="ras301") to see available regions. niveau 1=5 regioner, niveau 2=11 landsdele, niveau 3=99 kommuner.
- branche07 uses DB07-37-group letter codes (A, B, CA, CB...S, X = 37 sectors). Does NOT join dim.db. Use ColumnValues("ras301", "branche07") — labels are self-contained. All codes are at the same level (no hierarchy to worry about).
- socio does NOT join dim.socio — RAS codes 5, 10, 15, 20, 25, 30, 35, 40. No total row.
- koen (note: koen not kon) has only M and K — no total. Sum M+K for totals.
- alder has no total row and no TOT code.
- Map: /geo/kommuner.parquet (niveau 3), /geo/landsdele.parquet (niveau 2), or /geo/regioner.parquet (niveau 1) — merge on omrade=dim_kode. Exclude omrade IN (0, 950).
