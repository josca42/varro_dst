table: fact.ras300
description: Beskæftigede (ultimo november) efter branche (DB07), socioøkonomisk status, alder, køn og tid
measure: indhold (unit Antal)
columns:
- branche07: join dim.db on branche07=kode::text; levels [2, 3, 5]
- socio: join dim.socio on socio=kode
- alder: values [-15=Under 16 år, 16-19=16-19 år, 20-24=20-24 år, 25-29=25-29 år, 30-34=30-34 år, 35-39=35-39 år, 40-44=40-44 år, 45-49=45-49 år, 50-54=50-54 år, 55-59=55-59 år, 60-64=60-64 år, 65-66=65-66 år, 67-=67 år og derover]
- kon: values [TOT=I alt, M=Mænd, K=Kvinder]
- tid: date range 2008-01-01 to 2023-01-01
dim docs: /dim/db.md, /dim/socio.md
notes:
- branche07 uses DB07-127 coding with two embedded hierarchy levels: single/double-digit codes (1, 2, 3...11) are sector totals, 5-digit codes (01000, 02000, 10001...) are detailed sub-sectors. TOT = all industries. Never mix both levels — filter to either the sector level OR the sub-sector level. Use ColumnValues("ras300", "branche07") to browse with labels.
- socio does NOT join dim.socio — uses RAS-specific codes 5, 10, 15, 20, 25, 30, 35, 40 (employed persons only). Labels: 5=Selvstændige, 10=Medarbejdende ægtefæller, 15=Lønmodtager med ledelsesarbejde, 20=Højeste niveau, 25=Mellemniveau, 30=Grundniveau, 35=Andre lønmodtagere, 40=U.n.a. No total row — sum all 8 codes for total employment.
- alder has no total row: 13 age groups from -15 (under 16) to 67+. Sum explicitly.
- kon has TOT, M, K.
- To count total employed: filter kon='TOT', pick one branche07 level (single-digit for main sectors), sum all 8 socio codes.
