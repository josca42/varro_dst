table: fact.fra020
description: Fravær efter sektor, køn, fraværsårsag, arbejdsfunktion, fraværsindikator og tid
measure: indhold (unit -)
columns:
- sektor: values [1000=Sektorer i alt, 1032=Offentlig forvaltning og service, 1016=Stat (inklusiv sociale kasser og fonde), 1018=Kommuner og regioner i alt, 1020=Regioner, 1025=Kommuner, 1046=Virksomheder og organisationer, 1047=Virksomheder og organisationer - fastlønnede, 1048=Virksomheder og organisationer - timelønnede]
- kon: values [0=I alt, 1=Mænd, 2=Kvinder]
- fravaer: values [1100=Egen sygdom, 1200=Børns sygdom, 1300=Arbejdsulykke, 1400=Barsels- og adoptionsorlov]
- arbf: join dim.disco on arbf=kode::text [approx]; levels [1, 2, 3]
- fravaer1: values [10=Fraværsprocent, 40=Gennemsnitlige antal fraværsdagsværk pr. fuldtidsansat, 50=Gennemsnitlige antal fraværsperioder pr. helårsansat, 60=Gennemsnitlige antal kalenderdage pr. fraværsperiode, 70=Fuldtidsansatte, 80=Helårsansatte, 30=Fraværsperioder (antal), 20=Fraværsdagsværk (antal)]
- tid: date range 2013-01-01 to 2023-01-01
dim docs: /dim/disco.md
notes:
- fravaer1 is a measurement-type selector with 8 values (fraværsprocent, fraværsdagsværk pr. ansat, fraværsperioder pr. ansat, kalenderdage pr. periode, fuldtidsansatte, helårsansatte, fraværsperioder antal, fraværsdagsværk antal). Every dimension combination is repeated 8 times. Always filter fravaer1 to one value to avoid nonsense sums or doubled counts.
- fravaer is the absence cause (4 values). Filter unless you want all causes combined. sektor='1000' and kon='0' are the totals.
- sektor='1000' is total across all sectors — including it alongside sector codes causes double-counting.
- arbf joins dim.disco. The column mixes niveau 1 codes (single digit 0-9) and niveau 2 codes (two digit 11-96). Pick one niveau: WHERE d.niveau=1 for 10 major groups, d.niveau=2 for 42 sub-groups.
- Unmatched arbf codes: '01','02','03' are zero-padded military sub-group codes (dim.disco has these as 110, 210, 310 at niveau 4); '9999'=Uoplyst; 'TOT'=I alt. These have no dim entry.
- Use ColumnValues("disco", "titel", for_table="fra020") to see the 49 disco codes present in this table.
