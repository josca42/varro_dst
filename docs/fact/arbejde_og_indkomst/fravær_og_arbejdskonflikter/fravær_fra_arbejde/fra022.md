table: fact.fra022
description: Fravær efter sektor, køn, fraværsårsag, branche, fraværsindikator og tid
measure: indhold (unit -)
columns:
- sektor: values [1000=Sektorer i alt, 1032=Offentlig forvaltning og service, 1016=Stat (inklusiv sociale kasser og fonde), 1018=Kommuner og regioner i alt, 1020=Regioner, 1025=Kommuner, 1046=Virksomheder og organisationer, 1047=Virksomheder og organisationer - fastlønnede, 1048=Virksomheder og organisationer - timelønnede]
- kon: values [0=I alt, 1=Mænd, 2=Kvinder]
- fravaer: values [1100=Egen sygdom, 1200=Børns sygdom, 1300=Arbejdsulykke, 1400=Barsels- og adoptionsorlov]
- erhverv: join dim.db on erhverv=kode::text [approx]; levels [2, 3]
- fravaer1: values [10=Fraværsprocent, 40=Gennemsnitlige antal fraværsdagsværk pr. fuldtidsansat, 50=Gennemsnitlige antal fraværsperioder pr. helårsansat, 60=Gennemsnitlige antal kalenderdage pr. fraværsperiode, 70=Fuldtidsansatte, 80=Helårsansatte, 30=Fraværsperioder (antal), 20=Fraværsdagsværk (antal)]
- tid: date range 2013-01-01 to 2023-01-01
dim docs: /dim/db.md
notes:
- erhverv contains two parallel coding systems mixed in one column: numeric codes (1-11) are the 10-group DB aggregation matching dim.db_10 niveau 1, and letter codes (A, B, C, D, E, F, G, H, I, J, K, L, M, N, O, P, Q, R, S, X) are DB07 sections. Both levels exist simultaneously — picking one or the other avoids double-counting. The dim.db join noted above is incorrect (the numeric codes falsely match dim.db niveau 2 with wrong meanings); do NOT join erhverv to dim.db.
- Use ColumnValues("fra022", "erhverv") to get all labels. Example: '1'='1 Landbrug, skovbrug og fiskeri' (aggregate of A), 'A'='A Landbrug, skovbrug og fiskeri' (same sector at DB07 section level). For detailed breakdown use letters; for aggregated view use numbers. Never mix both in the same query without a WHERE filter.
- fravaer1 is a measurement-type selector (8 values) — always filter to one value.
- fravaer is absence cause (4 values). sektor='1000' and kon='0' are totals.
