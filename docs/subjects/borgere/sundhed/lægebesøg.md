notes:
**Table naming pattern:**
- sygk = Lægebesøg (visit counts), sygp = Personer (distinct person counts), sygu = Udgifter (costs, 1.000 kr.), sygs = specialty/detailed variants
- Suffix "1" = regions + communes (nuts niveau 1+3), broad age groups; suffix "2" = regions only (nuts niveau 1), single-year ages
- "ks"/"ps"/"us" = detaljeret (detailed specialist breakdown), "k"/"p"/"u" = grouped visit categories

**Choosing between tables:**
- For national/regional visit counts by visit type: **sygk1** (5-year ages, regions+communes) or **sygk2** (single-year ages, regions only)
- For persons with contact rather than visit counts: **sygp1/sygp2** (same geography/age split as sygk1/sygk2)
- For specialist breakdown (dermatology, cardiology, etc.): **sygks1/sygks2** (visit counts) or **sygps1/sygps2** (person counts)
- For public expenditure by visit type: **sygu1/sygu2** (grouped) or **sygus1/sygus2** (detailed specialties)
- For socioeconomic breakdown (rates, not counts): **sygher** (herkomst, 2006+), **sygind** (income quartiles, 2006+), **sygudd** (education, 2021+), **sygarb** (labour market, 2021+)
- For GP visits by family type: **ligehb6** (counts) or **ligehi6** (gender equality indicator)
- For children's GP contact by parental situation: **paaroe30** (0-17, 2021+)
- For adults' GP contact by social network: **paaroe31** (18+, 2021+)
- For health insurance group population: **sygsik** (commune level, through 2025)
- For prescription medicine: **medi1a** (single-year ages, region-only), **medi1b** (10-year ages, regions+communes), **medicin2/3** (by herkomst/education), **medicin4** (rates per 100k) — all experimental, 2016-2021 only

**Common pitfalls (all syg*/sygu*/sygs* tables with omrade):**
- omrade=0 is the national total; not in dim.nuts — exclude when joining
- omrade=86 (region-level tables) and omrade=98 (commune-level tables) are uoplyst — exclude when joining
- ydelsesart is hierarchical: 120=I ALT contains everything. Do NOT sum all ydelsesart values

**bnogle selector tables (sygudd, sygarb, sygher, sygind, paaroe30, paaroe31):**
- bnogle switches between two independent metrics (average contacts per person vs % with contact). Always filter to one bnogle value — never aggregate across both.

**Medicine tables (medi1a, medi1b, medicin2, medicin3, medicin4):**
- All experimental statistics, data only 2016-2021
- bnogle switches between person count and prescription count
- medicintype has two-level ATC hierarchy — do not mix levels
- medicin4 contains rates (per 100.000), not counts — do not sum

**Map availability:**
- Kommune-level choropleth (niveau 3): sygk1, sygks1, sygp1, sygps1, sygu1, sygus1, ligehb6, ligehi6, medi1b, sygsik — use /geo/kommuner.parquet, merge on omrade=dim_kode.
- Region-level choropleth (niveau 1): all of the above plus sygk2, sygks2, sygp2, sygps2, sygu2, sygus2, medi1a — use /geo/regioner.parquet.
