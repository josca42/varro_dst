notes:
- Most bib tables (bib1–bib8, sbs1) cover folkebiblioteker from 2009; ibib/fstrib tables start from 2020/2024 with individual-level demographic breakdowns.
- Table naming pattern: tables ending in 'a' (ibib1a, ibib2a, fstrib1a, etc.) are fixed annual archives (to 2024); the non-a equivalents are live tables updated through the current year — prefer non-a for current analysis.
- For aggregate national/regional library KPIs (loans, holdings, spending, staff): use bib1 (by metric), bib2b (service locations/folketal), bib6 (økonomi), bib8 (årsværk). These all cover 2009–.
- For individual material type breakdown: bib3 (physical materials, absolute counts), bib3b (physical materials, age-share percentages), bib4e (electronic resources), bib5 (interlibrary loans).
- For cross-municipality comparison with normalised rates: sbs1 (niveau 2 landsdele + niveau 3 kommuner; ~40 per-1000-inhabitant KPIs). Note: sbs1 has no niveau 1 regions.
- For demographic breakdown of borrowers (who uses libraries): ibib1 (by gender), ibib2 (by age), ibib3 (by education), ibib4 (by distance from home to library). All go from 2020 to 2025.
- For Filmstriben (online film streaming) specifically: fstrib1/2/3 — same demographic breakdown structure as ibib, but starting only from 2024.
- For school libraries: fsbib1 (folkeskolebiblioteker), only 2022–2024, kommune level only.
- For laby42: FTE per 1000 inhabitants by kommunegruppe type (Hoved/Storby/Provins/Opland/Land) from 2016. Use for comparing staffing intensity across municipality types.
- For survey/attitudinal data (purposes, digital services, barriers): kv2bib1/2/3 — all 2024 only, no geography, values are independent percentages (not mutually exclusive — never sum).
- Common pitfall: omrade=0 (hele landet) is the national aggregate and appears in most bib/ibib tables but is NOT a kode in dim.nuts. Use WHERE omrade='0' directly, not via a dim join.
- Common pitfall: many tables have a metric-selector column (bnogle, opgoer1, aktivi, dyrkat, misbrug2). Always filter to a single value or you will sum incompatible metrics.
- Map: most tables support choropleth maps. bib1/bib2b/bib3/bib3b/bib4e/bib5/bib6/bib8 use omrade (niveau 1 regioner, niveau 3 kommuner); sbs1 uses omrade (niveau 2 landsdele, niveau 3 kommuner); ibib1–4 and 'a' variants use omrade (all 3 levels); fsbib1 uses kommunedk (niveau 3 only); fstrib1–3 and 'a' variants use kommunedk (all 3 levels). kv2bib1/2/3 and laby42 have no geographic column.
