notes:
- For total material flow accounts (MFA): use mrm2 (1993–2023, in tonnes). For resource footprint in raw material equivalents: use rme1 (2008–2023, in 1.000 tonnes). For headline DMC indicator (per capita, per BNP): use sdg08042.
- mrm2 and rme1 both require filtering indikator to one flow type before summing.

- Waste production by industry: affald (3 dimensions: erhverv+behandling+afffrak), affald01 (no behandling), affald02 (no afffrak), affald03 (by hazard), affald04 (import/export only). All cover 2011–2023.
- Household waste by kommunegruppe/municipality: laby24 (absolute tonnes, 2013–2022), laby25 (recycling rate % and kg/person, 2013–2022). Both use dim.kommunegrupper: niveau 1=5 kommunegrupper, niveau 2=99 kommuner. komgrp=0 is aggregate total and won't join.

- Waste multiplier tables (economic attribution): aff1mu*n=by fraction, aff2mu*n=by treatment, aff3mu*n=by hazard; mu1=by branche, mu2=by endelig anvendelse, mu3=by branche+origin. All cover 2011–2023. Always filter mult to one measurement type (DIR/INT/MUL) and prisenhed to one price basis.
- In aff*mu1n and aff*mu3n, branche joins dim.nr_branche via REPLACE(branche,'V','')=kode, but ~44% of 6-digit sub-industry codes don't appear in the dim. branche='V' is the aggregate total.

- Detailed material flow accounts (msr* series): 9 tables combining 3 detaljeniveauer (1=broad, 2=mid, 3=detailed mater codes) × 3 flow directions (o=oversigt, t=tilgang/inflow, a=anvendelse/outflow). All limited to 2018–2020 only. Use msr3t/msr3a for the most granular material breakdown. Never use the 'o' tables for summing — they contain both inflow and outflow totals.

- Common pitfall across all tables: hierarchical columns (mater, rastoftype, erhverv, afffrak, dyrkat) contain both aggregate and leaf codes. Summing all values in any of these columns will massively overcount. Always filter to a consistent level or use the total-row code only.
