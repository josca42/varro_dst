## Cross-cutting gotchas

- **tal selector (critical)**: All lbesk20+ tables have a `tal` column with two values: 1020=Lønmodtagere (headcount) and 1010=Fuldtidsbeskæftigede lønmodtagere (full-time equivalents). ALWAYS filter to one value — forgetting it silently doubles all counts. lbesk01–04 do NOT have this column.
- **sektor join is broken**: All tables with `sektor` as a dim.esr_sekt join are incorrectly documented. sektor uses 4-digit numeric codes (DST internal), not the letter codes in dim.esr_sekt. Treat sektor as inline in all tables.
- **branche column coding**: The `branchedb071038` column mixes two schemes in one column — numeric 1–11 (10-gruppe) and letter codes A–S (19-gruppe). Never aggregate across both. The `branchedb0738` column is 19-gruppe only (letter codes). Neither set of letter codes is in any dim table.
- **herkomst subcategories**: Tables with herkomst use codes 1/24/25/34/35/TOT/UDK — NOT the 4-code dim.herkomst taxonomy. 24=Indvandrere vestlige, 25=Indvandrere ikke-vestlige, 34=Efterkommere vestlige, 35=Efterkommere ikke-vestlige. Treat as inline.
- **geografisk hierarchy mixing**: arbejdslands, boplandk, and bopkom columns often contain both regioner (81–85) AND landsdele (1–11) in the same column. Filter by code range to pick one level.

## Choosing the right table

- **National totals by branche (10-group)**: lbesk01 (ukorr.) or lbesk03 (sæsonkorr.) — simple, no tal column, monthly back to 2008.
- **National totals by sektor only**: lbesk02 (ukorr., 2 sectors) or lbesk04 (sæsonkorr.) for quick offentlig/privat split. lbesk21/lbesk23 for the full stat/regioner/kommuner/sociale kasser breakdown.
- **Branche + enhed (lønmodtagere vs fuldtid)**: lbesk20 (ukorr.) or lbesk22 (sæsonkorr.). More flexible than lbesk01 — lets you choose headcount vs FTE.
- **Sæsonkorrigeret series**: lbesk03, lbesk04, lbesk22, lbesk23, lbesk30, obesk3, obesk4 are seasonally adjusted. Use these for trend analysis; use ukorrigerede for levels.
- **Regional breakdown (arbejdssted)**: lbesk30 (region+landsdel, no branche), lbesk33 (region+sektor), lbesk31 (region+branche, BUT values are pct. shares not counts), lbesk63 (region+kon+alder), lbesk64 (region+kon+herkomst).
- **Regional breakdown (bopæl)**: lbesk44 (region+kon+alder), lbesk45 (region+kon+herkomst), lbesk67 (region+sektor+kon+alder), lbesk68 (region+branche+kon+alder).
- **Kommune-level**: lbesk69 — only table with municipality-level granularity. bopæls-based. Has all NUTS levels mixed in one column.
- **Herkomst breakdown**: lbesk43 (national, branche+kon+alder), lbesk45 (bopæl+kon), lbesk62 (branche+sektor+kon), lbesk64 (arbejdssted+kon). All use vestlig/ikke-vestlig subcategories.
- **Immigrants only (lbesk46)**: lbesk46 covers indvandrere by country of origin (39 origin countries + aggregates). oprindland uses DST 3-letter codes — NOT joinable to dim.lande_uhv.
- **Public sector (obesk)**: obesk1/obesk4 = by COFOG purpose (10 categories), cofog dim join works. obesk2/obesk3 = by sector (stat/regioner/kommuner). All obesk = fuldtidsbeskæftigede only, no tal column.
- **Most recent data**: lbesk01–04 and lbesk30 go to 2025-08 (most recent). lbesk60–69 and some lbesk6x end at 2025-01.
- **lbesk31 is pct., not count**: The only table measured in "Andel i pct." — gives regional branch-share percentages, not headcounts. Do not sum across regions.
- **Map support**: 10 tables have geographic columns joinable to geo files. Arbejdsstedslandsdel (lbesk30, lbesk31, lbesk33, lbesk63, lbesk64) and bopælslandsdel (lbesk44, lbesk45, lbesk67, lbesk68) support regioner/landsdele choropleths. lbesk69 (bopælskommune) is the only table with kommune-level map granularity — use /geo/kommuner.parquet.
