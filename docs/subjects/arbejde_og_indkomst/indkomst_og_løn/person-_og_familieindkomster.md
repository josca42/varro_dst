notes:
- Tables split into person (indkp*, aindk*) and family (indkf*) series. Match to the unit of analysis first.

**Picking a person income table:**
- National totals by age/gender/income type → indkp201 (from 1996, most control dimensions)
- By kommune/landsdel, any income type → indkp101 (from 1987, levels 2+3)
- By region only, with income brackets → indkp102 (indkomst i alt) or indkfpp3 (disponibel)
- By landsdel + socioøkonomisk status → indkp104 (from 1994)
- By landsdel/region + age breakdown → indkp111 (from 1992, fine age groups) or indkp109 (from 2008, + herkomst)
- By area + herkomst → indkp109 only (levels 1+2, coarse age groups, 2008+)
- By area + education level → indkp107 (from 2004, all nuts levels)
- Income distribution (brackets), national → indkp221 (fine 50k steps, from 1995)
- Income distribution (brackets), by area → indkp222 (landsdele+kommuner) or indkp105/indkp106 (25k steps)
- A-income (employer-reported) only → aindk1/aindk2/aindk3 (all from 2012 only)

**Picking a family income table:**
- National totals by family type/income type → indkf201 (from 2004)
- Family counts by area → indkf11a (from 1987, all nuts levels)
- By area + income type → indkf111 (from 1991, all nuts levels, extended famtyp)
- By area + number of children → indkf112 (from 1993)
- By area + owner/renter → indkf101 (levels 2+3) or indkf102 (levels 1+2, with brackets)
- By area + socioøkonomisk status → indkf104 (from 1993)
- By area + education → indkf107 (from 2000)
- Income distribution by kommune → indkf122 (indkomst i alt) or indkf132 (disponibel)

**Cross-cutting gotchas:**
- enhed is a mandatory measurement selector in almost every table. Failing to filter it multiplies row counts (4-5x). Always filter enhed to a single value.
- indkomsttype is hierarchical in all tables that have it. Never sum across multiple types. The available types differ slightly between person tables (100-290 series) and aindk tables (11000-11500 series) and ligeii tables (IND1-IND7 series).
- All nuts-joined columns have a '0' code for the national aggregate that is not in dim.nuts. When joining to dim.nuts, '0' rows are silently excluded — use WHERE omrade='0' to get national totals.
- Tables with multiple nuts levels (indkp107, indkf107, indkf111, indkf112, indkf11a): filter WHERE d.niveau=... when joining, or results double-count across geographic levels.
- Family tables with indkintb: person tables use '0' for total bracket; family tables (indkf102, indkf122, indkf132) use '99' for total.
- socio tables (indkp104, ligeii1, indkf104): codes 100, 115, 130 exist in the fact table but not in dim.socio. INNER JOIN silently drops them.
- ligeii tables (ligeii1, ligeii2): indikator column has LAK1/LAK2/LAK3 — these are not summable (two are kr values, one is a percentage). Always filter to one indikator.
- laby13 and ligeii4 show percentages — never sum indhold across categories.
- Map: most tables support choropleth maps via dim.nuts.
