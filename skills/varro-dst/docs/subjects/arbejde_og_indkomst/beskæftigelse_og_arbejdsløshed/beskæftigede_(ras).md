notes:
**Table series:**
- ras3xx = snapshot ultimo november (employed on the last day of November). kas3xx = annual/quarterly average (gennemsnitligt antal). Choose ras for point-in-time counts, kas for averages over a period.
- laby29/30/31 = municipal group (kommunegruppe) breakdowns for wage earners.
- ligeab/ligeai/ligefi = gender equality indicators — ligeab gives counts by gender, ligeai gives gender ratio percentages, ligefi gives average working hours by gender.

**branche07 coding schemes vary across tables — critical:**
- DB07-37-group (letter codes A, B, CA, CB...S, X = 37 sectors): ras301, ras302, ras307, ras308, ras310, kas301, kas307, kas308, ligeab4, ligeab7, ligeab8, laby29, laby31, ligeai4, ligeai5. No hierarchy — all codes at the same level. Does NOT join dim.db.
- DB07-127-group (5-digit codes 01000, 02000...): ras300, ras311, kas300, kas311, ligedb9, ligedi9, kas309 (actually 6-digit). Has two hierarchy levels — single/double-digit codes are sector totals, 5-digit are sub-sectors. Never mix both levels.
- DB07-detailed (6-digit codes 011100, 101110...): ras309, ligeai3, kas309. Most granular, mostly matches dim.db niveau 5. Use ColumnValues for labels.
- Public-sector-only (84001-88000): laby30. Only covers public forvaltning og service branches.

**socio column (ras300, ras301, ras302, ras308, kas300, kas301, kas308):**
Does NOT join dim.socio. Uses RAS-specific codes 5, 10, 15, 20, 25, 30, 35, 40. No total row — sum all 8 codes for total employment. See ColumnValues.

**sektor column (ras305, ras307, kas305, kas307, laby29, laby30):**
Does NOT join dim.esr_sekt. Inline codes 1015=Stat, 1020=Regioner, 1025=Kommuner, 1030=Sociale kasser og fonde, 1035=Offentlige virksomheder, 1040=Private virksomheder, 1045=Private nonprofit-organisationer, 1050=Uoplyst. Most tables lack a total sector code (except ligeab7 code 1000, kas307 code 1000).

**omrade/arbejdssted (ras301, ras302, ras305, ras310, kas301, kas305, ligeab4):**
Joins dim.nuts at ~98%. Code 0=Hele landet and 950=Uden for Danmark are aggregate codes not in dim.nuts. niveau 1=5 regioner, niveau 2=11 landsdele, niveau 3=99 kommuner.

**periode column (all kas tables):**
Always filter to one value. 1=Hele året (annual average), K1-K4=quarterly averages. Annual average ≠ sum of quarterly.

**Percentage tables — never sum indhold:**
laby29a, laby29b, laby30a = shares in percent. ligeai3, ligeai4, ligeai5, ligeai7, ligeai8, ligedi9 = gender ratio indicators (not counts). ligefi5 = average hours (not counts).

**Which table to use:**
- Employed persons by sector + socioeconomic status: ras300 (national) or ras301 (regional, letter-coded sectors)
- Employed persons by region + detailed sector: ras302 (same as ras301 but kon/koen column naming)
- Wage earners by workplace region + full/part-time: ras305 or kas305 (average)
- By origin/herkomst: ras308, ras311, kas308, kas311, ligeab8, ligeai8
- Subsidised employment (støttet beskæftigelse): ligeab5, ligeai5
- Insurance category (forsikringskategori): ligeab4, ligeai4
- Family type breakdown: ligeab7, ligeab8, ligeai7, ligeai8, ligefi5
- Most detailed sector classification: ras309, kas309, ligeai3
- Public sector by municipality group: laby30, laby30a
- Private sector by municipality group: laby31
- All sectors by municipality group: laby29, laby29a, laby29b

**Map support:**
- Tables with regional breakdown (ras301, ras302, ras310, kas301, ligeab4) support choropleth maps via omrade=dim_kode using /geo/kommuner.parquet (niveau 3), /geo/landsdele.parquet (niveau 2), or /geo/regioner.parquet (niveau 1).
- Tables with workplace area breakdown (ras305, kas305) support the same geo files via arbejdssted=dim_kode.
