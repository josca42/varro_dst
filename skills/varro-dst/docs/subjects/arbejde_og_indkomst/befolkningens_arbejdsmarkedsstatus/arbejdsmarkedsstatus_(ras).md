## Choosing a table

**Annual (RAS) vs quarterly (KAS):**
- RAS tables (ras200–ras210) = annual snapshots measured ultimo november, 2008–2023.
- KAS tables (kas200–kas209) = quarterly counterparts, 2017–2023. They add a `periode` or `tiden` column (1=Hele året, K1–K4). Always filter this column — forgetting inflates counts 5×.
- Use RAS for longer time series (back to 2008) or when quarterly granularity isn't needed. Use KAS for seasonal/quarterly analysis from 2017.

**Frequency rates (pct.) vs headcounts (antal):**
- Employment and labor force participation rates: ras200/kas200 (by region + herkomst), ras204/kas204 (by country-of-origin)
- Gender equality indicators (rates + gender gap): ligeai1, ligeai2
- All other tables: headcounts

**Which breakdown do you need?**
- Regional (omrade, dim.nuts): ras200, ras201, ras209, ras210, ligeab2, ligeai2, kas200, kas201, kas209, flow2
- Country-of-origin (oprindland): ras204, ras205, ras206, ras208, kas204, kas205, kas208
- Education level (uddannelse): ras209, kas209
- Family type (famtype): ligeab1, ligeab2, ligeai1, ligeai2
- Status transitions between quarters: flow1 (national, detailed socio), flow2 (municipality level, simplified socio)
- Length of stay in Denmark (ophtid): ras206 only (immigrants only, 2008–2023)

**Socioeconomic status (socio) granularity:**
- Detailed 30+ categories: ras202, ras205, ras209, kas202, kas205, kas209
- Simplified 3 categories (Beskæftigede / Arbejdsløse / Uden for arbejdsstyrken): ras201, ras206, ras208, ras210, ligeab1, ligeab2, flow2, kas201, kas208

**ras205 vs ras208 (and kas205 vs kas208):** Both cover oprindland × socio × alder × kon, but ras205/kas205 = detailed socio + 5-year age groups, ras208/kas208 = simplified socio + single-year ages. Descriptions are identical — distinguish by socio column values.

## Cross-cutting gotchas

- `beregning` (BFK/EFK) in ras200, ras204, kas200, kas204 is a measurement selector. Always filter to one value.
- `omrade='0'` (and similar aggregate codes) is the national total — not in dim.nuts. Use directly without JOIN.
- dim.nuts has 115 codes: 5 regioner (niveau 1), 11 landsdele (niveau 2), 99 kommuner (niveau 3). Filter `WHERE d.niveau = N` to avoid summing across hierarchy levels.
- Several tables have NO kon total (only M and K): ras201, ras205, ras206, ras208, flow1, flow2. SUM M+K to get combined gender.
- `herkomst` coding is inconsistent across tables — ras200 uses 0/10/20/24/25/30/34/35, ras201 uses 999 for Dansk oprinde, kas201 uses 1. Check the specific table's values.
- ligeai1/ligeai2 `indikator` (LA1=Mænd%, LA2=Kvinder%, LA3=Forskel) — LA3 is a derived gender gap, not summable with LA1/LA2. Always filter to one indikator.
- flow1/flow2 `bevaegelsev` encodes distinct quarter-transitions (1-2KVT, 2-3KVT, 3-4KVT) — not duplicates.
- ras202/kas202 use `koen` (not `kon`) for gender. kas202/kas204/kas208 use `tiden` (not `periode`) for the time-granularity selector.
- Map: ras200, ras201, ras209, ligeab2, ligeai2, kas200, kas201, kas209 (omrade) and ras210, flow2 (bopkom) all support choropleth maps at kommune (niveau 3), landsdel (niveau 2), or region (niveau 1) level via /geo/kommuner.parquet, landsdele.parquet, or regioner.parquet — merge on omrade/bopkom=dim_kode.
