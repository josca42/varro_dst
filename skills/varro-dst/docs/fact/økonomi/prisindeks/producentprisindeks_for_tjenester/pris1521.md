table: fact.pris1521
description: Producentprisindeks for tjenester efter branche, enhed og tid
measure: indhold (unit Indeks)
columns:
- erhverv: join dim.db on erhverv=kode [approx]; levels [2, 3, 4]
- enhed: values [100=Indeks, 210=Ændring i forhold til kvartalet før (pct.), 310=Ændring i forhold til samme kvartal året før (pct.)]
- tid: date range 2006-01-01 to 2025-04-01
dim docs: /dim/db.md

notes:
- enhed is a measurement-type selector that triplicates every row: 100=basisindeks, 210=kvartalsvækst (pct.), 310=årsvækst (pct.). Always filter to exactly one enhed, e.g. WHERE f.enhed = 100. Forgetting this triples all counts.
- erhverv joins dim.db but spans three DB07 levels simultaneously: 8 two-digit niveau-2 codes (52, 62, 63, 69, 71, 73, 78, 80), 5 three-digit niveau-3 codes (494, 521, 522, 532, 812), and 13 four-digit niveau-4 codes (4941, 4942, 5221, 5224, 5229, 6820, 6832, 6910, 6920, 7022, 7111, 7112, 7120). Always filter the join by a single niveau — never join without d.niveau, as codes 52, 62, 71 and 812 each exist at two different niveaux in dim.db with completely different meanings (e.g. code 52 is "Hjælpevirksomhed i forbindelse med transport" at niveau 2 but "Indvinding af brunkul" at niveau 3).
- Parent-child codes coexist in the fact: code 52 (niveau 2) and its children 521, 522 (niveau 3) are all present. Summing across all erhverv values double-counts these sub-sectors. Pick one niveau at a time.
- Code 6970 is not in dim.db and will silently drop from any JOIN. It appears to be a DST custom code (229 rows, all three enhed values). Label it manually if needed: roughly "anden juridisk og bogholderimæssig virksomhed" under kode 69.
- Safe query pattern for the 8 top-level service categories (niveau 2):
  SELECT d.titel, f.tid, f.indhold
  FROM fact.pris1521 f
  JOIN dim.db d ON f.erhverv = d.kode AND d.niveau = 2
  WHERE f.enhed = 100
  ORDER BY d.titel, f.tid;
- For sub-sector queries, choose niveau 3 or 4 explicitly and be aware not all industries have sub-codes in this table (e.g. code 62 IT only appears at niveau 2, while code 52 transport has sub-codes at both niveau 3 and 4).
