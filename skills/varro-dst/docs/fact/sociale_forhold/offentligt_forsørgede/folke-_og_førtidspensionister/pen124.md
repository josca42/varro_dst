table: fact.pen124
description: Ydelse til folkepensionister med bopæl i Danmark efter køn, alder, ydelsestype, familietype, enhed og tid
measure: indhold (unit -)
columns:
- kon: values [MOK=Mænd og kvinder i alt, M=Mænd, K=Kvinder]
- alder: values [TOT=Alder i alt, 6569=65-69 år, 7074=70-74 år, 7579=75-79 år, 8089=80-89 år, 9099=90 år og derover]
- ydelsestype: values [46=Folkepension i alt, 47=Folkepension uden pensionstillæg (brøkpension undtaget), 48=Folkepension med fuldt pensionstillæg (brøkpension undtaget), 49=Folkepension med reduceret pensionstillæg (brøkpension undtaget), 50=Brøkpension til folkepensionister]
- famtype: values [FAMIALT=Familier i alt, ENL=Enlige, PAF=Parfamilier]
- enhed: values [7=Gennemsnitlig ydelse (kr.), 71=Alle modtagere (antal personer)]
- tid: date range 2021-01-01 to 2025-06-01

notes:
- enhed is a measurement selector — every dimension combination exists twice, once for each value. Always filter to a single enhed: 7=Gennemsnitlig ydelse (kr./måned), 71=Alle modtagere (antal). Summing across both enhed values is meaningless (adding kroner and person-counts).
- Use enhed=71 to get recipient counts (same total as pen114 modtag1=80, but without regional breakdown). Use enhed=7 to get average monthly benefit amount in DKK.
- famtype is an independent dimension: FAMIALT=total, ENL=enlige, PAF=parfamilier. Filter famtype='FAMIALT' for overall averages/counts.
- alder=TOT is the total; 6569, 7074, 7579, 8089, 9099 are age bands. ydelsestype=46 is the total; 47–50 are sub-types — do not mix.
- This table has no regional breakdown (unlike pen114). For national-level average benefit amounts, this is the right table.
