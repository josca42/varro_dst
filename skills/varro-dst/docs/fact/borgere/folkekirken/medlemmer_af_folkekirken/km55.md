table: fact.km55
description: Befolkningen 1. januar efter provsti, køn, alder, folkekirkemedlemsskab og tid
measure: indhold (unit Antal)
columns:
- provsti: values [101=1-01 Københavns Vor Frue Provsti, 102=1-02 Københavns Holmens Provsti, 103=1-03 Østerbro Provsti, 104=1-04 Nørrebro Provsti, 105=1-05 Vesterbro Provsti ... 1004=10-04 Fredericia Provsti, 1005=10-05 Kolding Provsti, 1006=10-06 Vejle Provsti, 1007=10-07 Hedensted Provsti, 9999=99-99 Uoplyst provsti]
- kon: values [1=Mænd, 2=Kvinder]
- alder: values [0=0 år, 1=1 år, 2=2 år, 3=3 år, 4=4 år ... 121=121 år, 122=122 år, 123=123 år, 124=124 år, 125=125 år]
- fkmed: values [F=Medlem af Folkekirken, U=Ikke medlem af Folkekirken]
- tid: date range 2007-01-01 to 2025-01-01

notes:
- Provsti-level equivalent of km5 (annual, age+gender+membership). Unlike km5 (5-year bands), alder here uses individual years 0–125 — 113 distinct values, no IALT total row.
- To get total population for a provsti, sum across all alder values: WHERE provsti='...' GROUP BY provsti, fkmed (no alder filter needed — just SUM(indhold)).
- For age group analysis, use CASE WHEN alder::int BETWEEN 0 AND 4 THEN '0-4' ... expressions in SQL.
- kon: 1=Mænd, 2=Kvinder. No TOT — sum both for combined total.
- fkmed: F=Medlem, U=Ikke-Medlem. Filter to one or sum both explicitly.
