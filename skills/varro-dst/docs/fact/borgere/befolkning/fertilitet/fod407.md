table: fact.fod407
description: Fertilitetskvotienter efter område, alder og tid
measure: indhold (unit Pr. 1.000 kvinder)
columns:
- omrade: join dim.nuts on omrade=kode; levels [1, 2, 3]
- alder: values [TOT1=Samlet fertilitet, 15-19=15-19 år, 20-24=20-24 år, 25-29=25-29 år, 30-34=30-34 år, 35-39=35-39 år, 40-44=40-44 år, 45-49=45-49 år]
- tid: date range 2006-01-01 to 2024-01-01
dim docs: /dim/nuts.md

notes:
- omrade joins dim.nuts across all 3 hierarchy levels: niveau 1 = 5 regioner, niveau 2 = 11 landsdele, niveau 3 = 94 kommuner. Always filter WHERE d.niveau = N to pick one level — otherwise rows from all three levels are returned and values are not additive.
- Code '0' (Hele landet) is a national aggregate not present in dim.nuts. Use omrade='0' directly (without joining dim.nuts) to get national totals.
- Use ColumnValues("nuts", "titel", for_table="fod407") to see the 110 region codes available in this table.
- alder column: TOT1 = total fertility rate, remaining values are 5-year age groups (15-19 through 45-49). Filter alder='TOT1' when you only want the summary rate. Do not sum age group rates — they are rates per 1.000 kvinder in each age group, not counts.
- Sample query — fertility by region for latest year:
  SELECT d.titel, f.indhold FROM fact.fod407 f JOIN dim.nuts d ON f.omrade=d.kode WHERE f.alder='TOT1' AND f.tid='2024-01-01' AND d.niveau=1 ORDER BY f.indhold DESC;
- Map: /geo/kommuner.parquet (niveau 3), /geo/landsdele.parquet (niveau 2), or /geo/regioner.parquet (niveau 1) — merge on omrade=dim_kode. Exclude omrade=0.
