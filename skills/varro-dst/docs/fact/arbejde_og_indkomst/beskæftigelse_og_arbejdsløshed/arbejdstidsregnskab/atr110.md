table: fact.atr110
description: Arbejdstidsregnskab (kvartal) efter branche (DB07), sektor, type, socioøkonomisk status og tid
measure: indhold (unit -)
columns:
- branche07: join dim.db on branche07=kode::text [approx]; levels [2, 3]
- sektor: values [TOT=I alt, 0=Virksomheder og organisationer, 1=Offentlig forvaltning og service]
- type: values [HAC=Præsteret tid (1000 timer), ABESK=Beskæftigede (antal), AJOB=Job (antal), WSUM=Lønsum (mio. kr.)]
- socio: join dim.socio on socio=kode::text [approx]; levels [2]
- tid: date range 2008-01-01 to 2025-04-01
dim docs: /dim/db.md, /dim/socio.md

notes:
- `type` is a measurement selector — every dimension combination appears 4x (once per type). Always filter to one: HAC=præsteret tid (1000 timer), ABESK=beskæftigede (antal), AJOB=job (antal), WSUM=lønsum (mio. kr.).
- `branche07` mixes TWO classification schemes in the same column: numeric codes 1-11 (DB07 10-gruppe: 10 aggregated groups + 11=uoplyst) and letter codes A-S plus X (DB07 19-gruppe: the 19 afsnit sections). Code 1 = A (landbrug identical), but code 2 aggregates B+C+D+E (industri etc.). Always filter to ONE scheme to avoid double-counting.
- Numeric codes 1-11 can join dim.db with `d.niveau = 2`. Code `11` (uoplyst) creates an ambiguous join — in dim.db `11` matches both niveau 2 ("Fremstilling af drikkevarer") and niveau 3 ("Dyrkning af etårige afgrøder"). In this table code `11` means unknown/other, not a real afsnit. Exclude it or handle separately.
- Letter codes A-S, X cannot join dim.db (dim.db uses numeric codes only). For labels on letter codes, use ColumnValues("atr110", "branche07") to see the inline mapping if available.
- `socio` has custom code `14` (not in dim.socio) — a combined wage earner category specific to this table. Only code `13` (Lønmodtagere) joins dim.socio at niveau 2. `TOT` is the aggregate.
- To get a single total (e.g., all employed in Q): filter `sektor='TOT', socio='TOT', branche07='TOT', type='ABESK'` → one row.
