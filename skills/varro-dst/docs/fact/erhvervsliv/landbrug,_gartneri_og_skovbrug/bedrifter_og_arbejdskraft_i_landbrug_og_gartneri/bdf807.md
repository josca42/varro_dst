table: fact.bdf807
description: Bedrifter efter område, beskæftigede uden for bedriften og tid
measure: indhold (unit -)
columns:
- omrade: join dim.landbrugslandsdele on omrade::text=kode [approx]; levels [3]
- beskland: values [600=Landmand har arbejde uden for bedriften som bibeskæftigelse, 610=Landmand har arbejde uden for bedriften som hovedbeskæftigelse, 615=Ægtefælle har arbejde uden for bedriften som bibeskæftigelse, 620=Ægtefælle har arbejde uden for bedriften som hovedbeskæftigelse, 625=Ægtefælle deltager i bedriftens arbejde]
- tid: date range 2007-01-01 to 2023-01-01
dim docs: /dim/landbrugslandsdele.md

notes:
- DIMENSION DOC IS WRONG: omrade does NOT join dim.landbrugslandsdele (zero-padded kodes). It joins dim.nuts using plain integer codes, identical to bdf11: niveau 1 (81-85=regioner), niveau 2 (4,7,8,9,10=5 landsdele), plus aggregates 0 (Hele landet) and 15 (unmatched). Use JOIN dim.nuts d ON f.omrade = d.kode.
- beskland has 5 independent categories — they are NOT mutually exclusive. The same farm (and its spouse) can contribute to several values simultaneously. Never sum across all beskland values; each represents a separate condition/question about the farm household.
- There is no "total" row in beskland. Each value stands alone as an independent count.
- Map: /geo/regioner.parquet (niveau 1, omrade 81-85) or /geo/landsdele.parquet (niveau 2, omrade 4,7,8,9,10 — 5 of 11 landsdele only) — merge on omrade=dim_kode. Exclude omrade IN (0, 15).
